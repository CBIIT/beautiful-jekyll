        git fetch upstream
        git merge upstream/master

Determination of GitHub or Helix

Note: Assume private unless it needs to be public:

  • (GitHub Pages?) or (People outside of BIDS-HPC need read access to the repo?)
    • Yes: public ----> GitHub
    • No: private: Definitely just a small # of people need access?
      • Yes ----> GitHub
      • No ----> Helix

My current feeling is that for any *private* repository that requires multiple users (e.g., specific project-related repositories like those for the DMI or GMB collaborations), we should not use GitHub, because if in the future we want to add more users, we'd be limited unless we upgrade.
